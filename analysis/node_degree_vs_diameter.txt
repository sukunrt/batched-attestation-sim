Graph diameter vs node count and mesh degree
=============================================

Random topologies from simctl/topology.py::generate_random_topology (the same
generator the attestation sims use), with no fanout nodes. For each
(N nodes, target degree D) we generate 10 graphs (seeds 0-9) and compute the
EXACT diameter via all-pairs shortest paths (scipy Dijkstra).

  hop diameter      = longest shortest-path in number of edges (unweighted)
  latency diameter  = longest shortest-path in milliseconds, weighting each edge
                      by its assigned inter-country latency (topology latency_ms)
  mean lat          = mean over all node pairs of the shortest-path latency (ms)
  avg deg           = realized average node degree (~= target D; node 0 is a
                      slight hub from the connectivity pass, so it can exceed D)

Values below are min / median / max across the 10 seeds.

Notes
-----
- Diameter grows logarithmically in N: 1000 -> 10000 (10x nodes) adds only ~+2
  hops at D=8 and ~+1 at D=12; latency diameter grows even slower (D=8: 356 ->
  412 ms, +16%).
- Mesh degree D matters far more than N. D=4 -> 12 roughly halves the hop
  diameter; the D=4 vs D=12 latency gap widens from ~100 ms at 128 nodes to
  ~300 ms at 10k (668 vs 364 ms) -- denser meshes pay off more at scale.
- Latency shrinks less than hops with higher D, because each hop is a real
  inter-country link (~50-100 ms) and crossing continents needs a few hops no
  matter how dense the graph is.
- The topology is not the propagation bottleneck: even at 10k nodes / D=8 the
  worst-case geographic path is ~0.4 s, vs a 12 s slot. The rest of the
  observed attestation latency is processing on top of geography (validation,
  ~20 ms publish ticks, batch windows, duplicate-signature traffic).
